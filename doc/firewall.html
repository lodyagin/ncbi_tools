NCBI Toolkit and Firewalls

NCBI is in the process of switching the method of accessing our databases from a custom protocol to standard http [hyper text transfer protocol]. If you access our databases from behind a security firewall or proxy, you should change your configuration to avoid problems with this release of the toolkit. If you do not know if you are behind a security firewall or proxy, please consult your network administrator. If you are not behind a firewall or proxy, or exclusively use a web browser to access the NCBI databases, you do not need to change your configuration to take advantage of these changes.

 

If you are behind a firewall or proxy:

To avoid performance problems and other issues, edit your NCBI configuration file (e.g. .ncbirc for unix and ncbi.ini on windows) to include the line "SRV_CONN_MODE=DISPATCHER" underneath the "[NET_SERV]" entry:

[NET_SERV]

SRV_CONN_MODE=DISPATCHER

 

What does this do?

When you first access NCBI databases, the connection is received by a server called a dispatcher. The purpose of the dispatcher is to connect your software to the appropriate database server. For example, if you are looking for article abstracts using Sequin, the dispatcher will connect Sequin to the PubMed server. This dispatcher has been rewritten to use http instead of a custom communications protocol. Because of unresolved issues, it is best at the present time to use the old dispatcher when behind a firewall or proxy, which is done by setting SRV_CONN_MODE=DISPATCHER. There is no need to use the old libraries with this setting, as the new libraries support both the old and new dispatchers.

 

Detailed description of configuration settings:

SRV_CONN_MODE -- specifies whether to use the old or new dispatcher and if the connection should be persistent or stateless. A stateless connection keeps no record of previous connections. While this is the standard way of using http, it is inefficient. Persistent http is more efficient, but creates problems as it is an uncommon way to use http. We hope to resolve these problems in a future version of the toolkit.

Values:

DISPATCHER –- use the old dispatcher

WWW -- use the new dispatcher via persistent http

STATELESS -- use the new dispatcher via stateless http

 

SRV_ENGINE_HOST –- the name of the dispatcher.

Value:

www.ncbi.nlm.nih.gov -- default (new dispatcher)

 

SRV_ENGINE_PORT –- which port on the dispatcher to use.

Value:

80 -- default

 

SRV_ENGINE_URL –- the url used to access the dispatcher, minus the server name.

Value:

/Service/nph-dispd.cgi -- default

 

SRV_PROXY_HOST -- If you are using the new http dispatcher through a proxy or firewall, set this to the name of the proxy or firewall. This will only work with a CERN-type proxy. If you do not know if you have a CERN-type proxy or firewall, please contact your network administrator.

Value:

not set –- default

 

SRV_PROXY_PORT: The port to use on the proxy or firewall

Value:

80 -- default
